# Parallel AI Analysis Template

**Instructions for AI Models**: 
You are being asked to contribute a parallel analysis of an existing GPT-4o conversation entry that has already been analyzed by other AI models (Claude, Grok, etc.). Your task is to provide your unique perspective while building upon existing analyses.

**Required Elements**:
1. **Model Attribution**: Clearly identify yourself (e.g., "GLM-4.6", "Claude Sonnet 4.5", etc.)
2. **Entry Reference**: Entry number and title
3. **Token/URL**: Include repository token and GitHub URL for verification
4. **Analysis Approach**: Brief 2-3 sentence excerpt of your analytical focus
5. **Parallel Perspective**: How your analysis differs from or complements existing ones
6. **Unique Contribution**: What new insight or framework you bring to the understanding

**Template Format**:
```
[Your 2-3 sentence excerpt from your analysis]

[Optional: Brief comparison with existing analyses if space permits]

---
**Processing Information**
AI Model: [Your Model Name]
Processing Date: [Date]
Session ID: [Session ID]
Raw Source: [Source file]
Entry Type: [Type]
Word Count: [Estimate]
Significance Score: [X/10]

## [Your Analysis Title]

[Your full analysis text]
```

**Important Notes**:
- Keep analysis concise but insightful
- Focus on what existing analyses missed or misunderstood
- Avoid simply repeating previous conclusions
- Bring your model's unique analytical strengths
- Maintain academic/therapeutic tone appropriate to content
- Include specific examples from conversation that support your points
- If you disagree with previous analyses, explain why respectfully

**Submission Instructions**:
1. Create analysis file following naming convention: `[entry]_[descriptive_title]_analysis_[your_model].md`
2. Push to GitHub repository using provided token
3. Update tracking files if applicable
4. Provide confirmation when complete
